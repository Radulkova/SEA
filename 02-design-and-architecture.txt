02-design-and-architecture

1. Тип на приложението
-Приложението представлява уеб базирана система за публикуване и разглеждане на автомобилни обяви.
То е реализирано като ASP.NET Core MVC приложение, което използва клиент-сървър архитектура. Потребителите достъпват системата чрез браузър, без нужда от инсталация


2. Структура на проекта
-Проектът е организиран по стандартната структура на ASP.NET Core MVC:
Controllers – съдържат логиката за обработка на заявките;
Models – описват данните (автомобили, поръчки, потребители);
Views – отговарят за визуализацията (страници);
Data – съдържа контекста към базата данни и конфигурации;
Services – съдържа бизнес логиката (напр. резервации);
wwwroot – статични файлове (снимки, CSS, JavaScript).


3. Основни модули / слоеве
-Приложението е разделено на следните основни модули:
Потребителски модул – регистрация, вход и управление на потребители;
Модул за автомобили – създаване, преглед и управление на обяви;
Модул за резервации – създаване и обработка на заявки за оглед;
Модул за търсене – филтриране по марка и модел;
Административен модул – управление на резервации и съдържание;
Data слой – връзка с базата данни чрез Entity Framework.


4. Основни класове / обекти / ентитита
-Основните обекти в системата са:
Car – представя автомобил (марка, модел, цена, описание, снимка);
Order (или Reservation) – представя заявка за оглед;
ApplicationUser – потребител на системата;
Brand – марка на автомобил;
Model – модел, свързан с конкретна марка.

-Тези класове се използват за съхранение и обработка на данни в приложението.



5. Таблици и връзки (ако има база данни)
-В базата данни съществуват следните основни таблици:
Cars – съдържа информация за автомобилите;
Orders – съдържа резервации;
Users – съдържа потребителите;
Brands – марки автомобили;
Models – модели автомобили.
Връзки:
Един Brand има много Models;
Един Model принадлежи към един Brand;
Един Car има един Model;
Един User може да има много Orders;
Един Order е свързан с един Car.

6. Диаграма / схема
-качила съм файл в репозиторито


7. Потребителски поток
-Основният поток на използване е следният:
Потребителят отваря началната страница;
Разглежда или търси автомобили;
Избира конкретен автомобил;
Влиза в профила си (ако не е логнат);
Създава резервация за оглед;
Администраторът обработва заявката.



8. Използвани технологии
C#
ASP.NET Core MVC
Entity Framework Core
SQL Server / LocalDB
HTML, CSS, JavaScript
Bootstrap



9. Защо са избрани тези технологии
-Избраните технологии са подходящи за разработка на уеб приложения:
ASP.NET Core MVC осигурява добра структура и разделение на логиката;
Entity Framework Core улеснява работата с базата данни;
SQL Server е надеждна система за съхранение на данни;
Bootstrap позволява бързо създаване на модерен дизайн;
C# е основният език, който се използва в .NET платформата и е подходящ за този тип приложения.
